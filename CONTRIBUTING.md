# Contributing

Changes should keep documentation, contracts, and shipped packages aligned.

## Workflow

1. Create a focused branch.
2. Update prose and, when applicable, the canonical API or package source first.
3. Run `npm run verify`.
4. Review generated redirects and machine-readable documentation.
5. Open a pull request with migration and compatibility notes for behavioral changes.

## Contract changes

Do not hand-edit an OpenAPI contract here to document behavior that the runtime does not implement. Update and review the canonical API contract, then run `npm run sync:contracts -- /path/to/teamgrid-api <full-api-commit-sha>` and commit the resulting contracts and `sources/contracts.json` together. The synchronizer deliberately reads from that Git object, so uncommitted API changes cannot be attributed to the recorded commit.

## Writing conventions

- Describe API v1 as the source of truth for new integrations.
- Describe API v0 as legacy compatibility.
- Keep examples free of real credentials, workspace data, and personal data.
- Prefer exact commands and complete endpoint URLs.
- State lifecycle status such as controlled beta or prerelease explicitly.
- Preserve old public URLs through `scripts/generate-redirects.mjs`.

## Generated files

`public/_redirects`, `public/llms.txt`, and `public/llms-full.txt` are generated by `npm run generate`. CI fails if committed generated output is stale.
