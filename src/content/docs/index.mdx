## Choose the interface that fits the job

<CardGrid>
  <Card title="API v1">
    The versioned, cell-aware controlled-beta contract for service integrations. Use scoped credentials, cursor pagination, idempotent writes, and signed webhooks.

    [Explore API v1](/api/v1/)
  </Card>
  <Card title="TypeScript SDK">
    A typed Node.js client with regional routing, bounded retries, timeouts, pagination iterators, and versioned errors.

    [Use the SDK](/sdk/)
  </Card>
  <Card title="CLI">
    The fastest way to inspect and automate TeamGrid from a terminal or CI job. It supports structured output and stable exit codes.

    [Use the CLI](/cli/)
  </Card>
  <Card title="MCP server">
    An optional local stdio adapter with a deliberately bounded read-only tool surface. It uses the same SDK and credentials as the CLI.

    [Configure MCP](/mcp/)
  </Card>
</CardGrid>

## Existing v0 integrations

API v0 remains available for compatibility. Its contract is frozen and documented separately, with migration guidance beside the legacy reference.

API v1 currently governs 182 operations across 112 paths. That number measures contract parity across
the HTTP API, SDK, and CLI—not complete TeamGrid product coverage. The
[capability ledger](/guides/capability-coverage/) records the remaining partial, planned, and private
areas explicitly.
